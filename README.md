# Determinantal Ideals in Lean 4

This repository is a Lean 4 / Mathlib formalization project on generic minors,
standard bitableaux, determinantal ideals, and Gröbner bases of determinantal
ideals.

The main ambient polynomial ring is the coordinate ring of a generic `m × n`
matrix:

```lean
MvPolynomial (Fin m × Fin n) k
```

The development formalizes the standard-bitableau route to the Gröbner basis
theorem for determinantal ideals, following the KRS correspondence, Swan's
straightening law, and Sturmfels' Hilbert-function argument.

## Current Status

The active development is no longer at the placeholder stage. The main
straightening and Gröbner-basis route is formalized in
`MyProject/Determinantalideals/Sturmfels.lean`.

Major completed results include:

```lean
theorem exists_krsEquiv
theorem straightening_law
theorem exists_standardBitableau_basis_determinantalRing
theorem hilbertFunction_detRing_eq_card_monomialExp_width_le
theorem theorem1_GrPlusOne_isGroebnerBasis_of_isAntidiagonalTermOrder

```

The current core determinantal-ideal files have no `sorry` or `admit`
placeholders.

## Formalized Components

### Generic Minors And Determinantal Ideals

The foundational API develops:

* the generic matrix over `MvPolynomial`;
* ordered row/column index data for minors, `MinorIndex`;
* generic minor polynomials, `genericMinor`;
* fixed-size minor sets and determinantal ideals;
* row/column content APIs for minor indices;
* finite and finite-generation results;
* evaluation at concrete matrices;
* compatibility under coefficient-ring homomorphisms and base change.

### Minor Terms And Raw Minors

The minor-term layer develops:

* diagonal, anti-diagonal, and permutation exponent vectors;
* signed permutation terms in determinant expansions;
* support, cardinality, and total-degree lemmas for determinant terms;
* the determinant expansion of a generic minor as a sum of permutation terms;
* raw minors `RawMinorIndex`, allowing arbitrary row/column maps;
* raw two-minor products `RawMinorPair` and their basic polynomial, content,
  sorting, sign, and slot APIs.

### Young Bitableaux, KRS, And Straightening

The main Sturmfels file develops:

* Young bitableaux and standard Young bitableaux;
* generalized permutations and monomial width;
* row insertion and reverse row insertion;
* forward and reverse KRS;
* the KRS equivalence between monomial exponent vectors and standard
  bitableaux;
* degree and width preservation under KRS;
* Swan-style filtered straightening for products of minors;
* linear independence and uniqueness of standard-bitableau expansions;
* the full straightening law.

### Gröbner Basis Route

The completed mathematical chain is:

```text
generic minors and anti-diagonal leading terms
  + KRS correspondence and Schensted width theorem
  + filtered straightening law
  + standard-bitableau basis and Hilbert-function comparison
  ⇒ Gröbner basis theorem for determinantal ideals
```

The final concrete theorem is:

```lean
theorem theorem1_GrPlusOne_isGroebnerBasis
```

for the concrete anti-diagonal lexicographic monomial order.

## Main Theorems

### KRS correspondence

```lean
theorem exists_krsEquiv
```

This packages the degree- and width-preserving correspondence between monomial
exponent vectors and standard Young bitableaux.

### Straightening law

```lean
theorem straightening_law
```

Every Young-bitableau polynomial has a unique finite expansion in standard
Young-bitableau polynomials, preserving degree and respecting the length
filtration.

### Standard-bitableau basis

```lean
theorem exists_standardBitableau_basis_determinantalRing
```

This constructs the standard-bitableau basis of the determinantal quotient.

### Hilbert-function and Gröbner-basis theorem

```lean
theorem hilbertFunction_detRing_eq_card_monomialExp_width_le
theorem theorem1_GrPlusOne_isGroebnerBasis_of_isAntidiagonalTermOrder
theorem theorem1_GrPlusOne_isGroebnerBasis
```

These connect the standard-bitableau basis and monomial-width count to the
anti-diagonal initial ideal and prove the Gröbner-basis theorem.

## Build

Assuming Lean 4, Lake, and the Mathlib cache are available:

```bash
lake exe cache get
lake build
```

The project currently builds successfully. Some unrelated upstream/local
`Groebner` warning messages may be emitted during the build.
