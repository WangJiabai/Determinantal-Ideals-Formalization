# Determinantal Ideals in Lean 4

[![Lean](https://github.com/WangJiabai/Determinantal-Ideals-Formalization/actions/workflows/lean.yml/badge.svg)](https://github.com/WangJiabai/Determinantal-Ideals-Formalization/actions/workflows/lean.yml)

This Lean 4 development formalizes the Gröbner-basis component of
Sturmfels’s theorem for determinantal ideals, together with a
coefficient-normalized reduced refinement. For a generic `m × n` matrix over
an arbitrary field, it proves that all `(r + 1) × (r + 1)` generic minors form
a Gröbner basis of the ideal they generate for every monomial order satisfying
`IsAntidiagonalTermOrder`. The unnormalized minors already satisfy the
interreduction condition. Multiplying each minor by the inverse of its
anti-diagonal leading coefficient yields a reduced Gröbner basis under the
imported Gröbner library’s monic definition of reducedness. The order
`antiDiagonalLex` is a concrete instance.

In Lean, `Jr m n r k` is by definition the ideal generated by these minors.
The development does not formalize the geometric identification of this
generated ideal with the vanishing ideal of the rank-`≤ r` matrix variety.
The reducedness result is stated for explicitly coefficient-normalized
generators; it does not assert that the unnormalized signed minors form a
monic family.

The ambient polynomial ring is the coordinate ring of the generic matrix:

```lean
MvPolynomial (Fin m × Fin n) k
```

The proof combines generic minors, diagonal and anti-diagonal term-order
infrastructure, the Knuth–Robinson–Schensted (KRS) correspondence, the
Doubilet–Rota–Stein straightening law via Swan’s Laplace-product proof, and a
degree-wise Hilbert-function argument. The final step combines unit rescaling
with a divisibility-rigidity argument to prove reducedness for the normalized
generators under the same class of anti-diagonal term orders.

## Source Layout

The following files lie on the transitive import path rooted at
[`ReducedGroebner.lean`](MyProject/Determinantalideals/ReducedGroebner.lean).

| Core theorem path | Role |
| --- | --- |
| [`DeterminantalIdeal.lean`](MyProject/Determinantalideals/DeterminantalIdeal.lean) | A project-local, Mathlib-style API for generic minors and the ideals they generate, based on `Matrix.mvPolynomialX`. |
| [`Basic.lean`](MyProject/Determinantalideals/Basic.lean) | Compatibility names and content lemmas for the paper-oriented development. |
| [`Finite.lean`](MyProject/Determinantalideals/Finite.lean) | Finiteness facts for minor indices and fixed-size minor sets. |
| [`Ideal.lean`](MyProject/Determinantalideals/Ideal.lean) | Generated-ideal membership and determinantal-ideal nesting. |
| [`MinorTerms.lean`](MyProject/Determinantalideals/MinorTerms.lean) | Diagonal, anti-diagonal, and permutation terms in determinant expansions. |
| [`DiagonalOrder.lean`](MyProject/Determinantalideals/DiagonalOrder.lean) | Abstract term-order predicates and concrete lexicographic orders. |
| [`Sturmfels_lemma.lean`](MyProject/Determinantalideals/Sturmfels_lemma.lean) | Project-local auxiliary lemmas used in the combinatorial and Hilbert-function proofs. |
| [`Bitableaux.lean`](MyProject/Determinantalideals/Bitableaux.lean) | Young bitableaux, generalized permutations, width, and `Jr`/`GrPlusOne`. |
| [`KRScorrespondence.lean`](MyProject/Determinantalideals/KRScorrespondence.lean) | Forward and reverse KRS constructions, the resulting equivalence, and degree/width compatibility. |
| [`StraighteningLaw.lean`](MyProject/Determinantalideals/StraighteningLaw.lean) | The Doubilet–Rota–Stein straightening development via Swan’s Laplace-product proof, linear independence of bitableau polynomials, and quotient bases. |
| [`Groebner.lean`](MyProject/Determinantalideals/Groebner.lean) | Hilbert-function comparison and the final abstract and concrete Gröbner-basis theorems. |
| [`ReducedGroebner.lean`](MyProject/Determinantalideals/ReducedGroebner.lean) | Interreduction of the original minors, coefficient normalization, and the abstract and concrete reduced Gröbner-basis refinements. |

The external `Groebner` package supplies the general Gröbner-basis API used
in the main theorem and its reducedness refinement.

## Main Results

The recommended descriptive declarations are arranged broadly by proof stage:
KRS and straightening lead to the quotient-basis and Hilbert-function results,
followed by the original Gröbner-basis theorem, interreduction, coefficient
normalization, and reducedness.

```lean
Determinantal.KRS.krsEquiv
Determinantal.straightening_law
Determinantal.exists_standardBitableau_basis_determinantalRing
Determinantal.hilbertFunction_detRing_eq_card_monomialExp_width_le
Determinantal.GrPlusOne_isGroebnerBasis_of_isAntidiagonalTermOrder
Determinantal.GrPlusOne_isGroebnerBasis_antiDiagonalLex
Determinantal.GrPlusOne_isInterreduced_of_isAntidiagonalTermOrder
Determinantal.normalizedGenericMinor
Determinantal.support_normalizedGenericMinor
Determinantal.leadingCoeff_normalizedGenericMinor
Determinantal.normalizedGrPlusOne
Determinantal.antidiagExp_le_permExp_imp_minorIndex_eq
Determinantal.normalizedGrPlusOne_isGroebnerBasis_of_isAntidiagonalTermOrder
Determinantal.normalizedGrPlusOne_isReduced_of_isAntidiagonalTermOrder
Determinantal.normalizedGrPlusOne_isReducedGroebnerBasis_of_isAntidiagonalTermOrder
Determinantal.normalizedGrPlusOne_isReduced_antiDiagonalLex
Determinantal.normalizedGrPlusOne_isReducedGroebnerBasis_antiDiagonalLex
```

Declarations beginning with `theorem1_`, `lemma5_`, and `lemma6_` are aliases
that retain the source paper’s numbering; users should prefer the descriptive
names. `Determinantal.exists_krsEquiv` is an existential paper-facing wrapper
around `Determinantal.KRS.krsEquiv`.

## Build

Fetch the dependency cache and build the repository with:

```bash
lake exe cache get
lake build
```

The Lake package is named `MyProject`, so imports take the following form:

```lean
import MyProject.Determinantalideals.ReducedGroebner
```

## Artifact audit

Run the artifact audit commands with:

```bash
bash scripts/check_sources.sh
lake env lean PaperAudit.lean
python3 scripts/check_axioms.py
bash scripts/check_artifact.sh
```

[`Audit.lean`](Audit.lean) uses `#print axioms` to display the transitive axiom
dependencies of selected core results. The presence of foundational constants
such as `Classical.choice`, `propext`, or `Quot.sound` does not indicate that
the project declares a project-specific axiom.
[`check_axioms.py`](scripts/check_axioms.py) runs this report and enforces the
explicit allowlist in
[`audit/axiom_allowlist.txt`](audit/axiom_allowlist.txt).

[`PaperAudit.lean`](PaperAudit.lean) compiles manuscript-facing declaration
references and selected theorem-signature checks.
[`check_sources.sh`](scripts/check_sources.sh) scans all tracked Lean sources,
including root audit files. The CI workflow runs the corresponding component
checks on every push and pull request. Source scanning, interface compilation,
kernel checking, and axiom-allowlist enforcement establish distinct
properties.

Generate manuscript metadata or fixed source links with:

```bash
python3 scripts/artifact_metadata.py --format markdown
python3 scripts/paper_links.py --format markdown
```

Publishable permalinks should be generated only from a clean, committed,
frozen revision; the link generator refuses to run on a dirty working tree.
