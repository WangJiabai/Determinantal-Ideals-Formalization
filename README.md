# Determinantal Ideals in Lean 4

This repository formalizes a Sturmfels-style proof that the `(r + 1) ×
(r + 1)` minors of a generic matrix form a Grobner basis for the determinantal
ideal `J_r` under an anti-diagonal term order.

The main polynomial ring is the coordinate ring of a generic `m x n` matrix:

```lean
MvPolynomial (Fin m × Fin n) k
```

The development combines generic minors, determinantal ideals, diagonal term
orders, the Knuth-Robinson-Schensted (KRS) correspondence, Swan's straightening
law, and the Hilbert-function argument from Sturmfels' theorem.

## Source Layout

Core files are under `MyProject/Determinantalideals/`.

| File | Role |
| --- | --- |
| `DeterminantalIdeal.lean` | Mathlib-style generic minors and determinantal ideals, based on `Matrix.mvPolynomialX`. |
| `Basic.lean` | Compatibility names for the original `Determinantal` API. |
| `Finite.lean` | Finiteness and `Fintype` facts for minor indices and fixed-size minor sets. |
| `Ideal.lean` | Determinantal ideal API beyond the first generic-minor layer. |
| `Eval.lean` | Evaluation of generic minors at concrete matrices. |
| `ConcreteMatrix.lean` | Concrete matrix minor compatibility under coefficient maps. |
| `BaseChange.lean` | Base-change compatibility for generic minors and determinantal ideals. |
| `Sign.lean` | Determinant sign lemmas used by the minor-term layer. |
| `MinorTerms.lean` | Diagonal, anti-diagonal, and permutation terms in determinant expansions. |
| `DiagonalOrder.lean` | Diagonal and anti-diagonal term-order predicates and concrete lex orders. |
| `Sturmfels_lemma.lean` | Auxiliary Mathlib-only lemmas used by the main proof. |
| `Bitableaux.lean` | Young bitableaux, standard bitableaux, generalized permutations, width, and `J_r`, `G_{r+1}`. |
| `KRScorrespondence.lean` | Forward and reverse KRS algorithms and the KRS equivalence. |
| `StraighteningLaw.lean` | Swan-style straightening, standard-bitableau bases, and the determinantal quotient basis. |
| `Groebner.lean` | Hilbert-function comparison and the final Grobner-basis theorem. |

## Main Results

The main public statements are:

```lean
Determinantal.KRS.krsEquiv
Determinantal.exists_krsEquiv
Determinantal.straightening_law
Determinantal.exists_standardBitableau_basis_determinantalRing
Determinantal.hilbertFunction_detRing_eq_card_monomialExp_width_le
Determinantal.theorem1_GrPlusOne_isGroebnerBasis_of_isAntidiagonalTermOrder
Determinantal.theorem1_GrPlusOne_isGroebnerBasis
Determinantal.GrPlusOne_isGroebnerBasis_of_isAntidiagonalTermOrder
Determinantal.GrPlusOne_isGroebnerBasis_antiDiagonalLex
```

Here `Jr m n r k` is the determinantal ideal generated by the `(r + 1) x
(r + 1)` minors, and `GrPlusOne m n r k` is the corresponding set of generic
minors.

## Build

Install cached dependencies and build the repository with:

```bash
lake exe cache get
lake build
```

The project uses the current package name `MyProject`, so imports are of the
form:

```lean
import MyProject.Determinantalideals.Groebner
```

## Audit Commands

Useful repository-level checks are:

```bash
rg "sorry|admit" .
rg "axiom" .
rg "unsafe" .
```

The formalization files in `MyProject/Determinantalideals` are intended to be
free of `sorry`, `admit`, project-specific axioms, and `unsafe` declarations.
The broad repository scans may also match documentation that describes these
audit commands.
