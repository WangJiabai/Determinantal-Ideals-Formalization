# AFM Artifact Guide

## Artifact scope

This artifact formalizes the Gröbner-basis component of Sturmfels’s theorem
for a generic matrix over an arbitrary field and its coefficient-normalized
reduced refinement. The main results accept every monomial order satisfying
`IsAntidiagonalTermOrder`; `antiDiagonalLex` is a concrete instance. The
original `(r + 1) × (r + 1)` generic minors form an interreduced Gröbner basis.
Multiplying each minor by the inverse of its anti-diagonal leading coefficient
gives a reduced Gröbner basis in the monic sense used by the Lean library. The
Lean definition `Jr` is the ideal generated by the original minors. The
artifact does not formalize the rank-variety geometry or the identification of
its vanishing ideal with `Jr`.

## Requirements

- Git;
- elan with Lake and the pinned Lean toolchain;
- network access only for the initial dependency/cache download;
- no external computer-algebra system as a proof oracle.

## Clean-checkout reproduction

Reviewers should use the immutable revision cited by the manuscript. Once that
revision has been frozen, a clean reproduction starts with:

```bash
git clone https://github.com/WangJiabai/Determinantal-Ideals-Formalization.git
cd Determinantal-Ideals-Formalization
git checkout ARTIFACT_COMMIT_OR_TAG
```

Replace `ARTIFACT_COMMIT_OR_TAG` with the immutable commit or tag cited by the
manuscript.

Then run:

```bash
lake exe cache get
bash scripts/check_artifact.sh
```

A successful run ends with `Artifact checks completed successfully`. It scans
tracked Lean sources, builds the project, compiles the paper-facing interfaces,
enforces the declared axiom allowlist, checks declaration locations used for
fixed links, and prints revision/dependency metadata. The initial cache command
may access the network; the proofs themselves do not call an external
computer-algebra system.

## Paper-facing entry points

- [`PaperAudit.lean`](PaperAudit.lean) compiles the declaration names and
  displayed theorem signatures used by the manuscript.
- [`Audit.lean`](Audit.lean) reports the axioms of selected declarations.
- [`Groebner.lean`](MyProject/Determinantalideals/Groebner.lean) contains the
  Hilbert-function argument and the unnormalized Gröbner-basis theorem.
- [`ReducedGroebner.lean`](MyProject/Determinantalideals/ReducedGroebner.lean)
  contains interreduction, coefficient normalization, and reducedness.
- Main Gröbner-basis theorem:
  `Determinantal.GrPlusOne_isGroebnerBasis_of_isAntidiagonalTermOrder`.
- Interreduction theorem for the original minors:
  `Determinantal.GrPlusOne_isInterreduced_of_isAntidiagonalTermOrder`.
- Divisibility rigidity lemma used by interreduction:
  `Determinantal.antidiagExp_le_permExp_imp_minorIndex_eq`.
- Main reduced-refinement theorem:
  `Determinantal.normalizedGrPlusOne_isReducedGroebnerBasis_of_isAntidiagonalTermOrder`.
- Reduced-refinement paper-number alias:
  `Determinantal.theorem1_normalizedGrPlusOne_isReducedGroebnerBasis_of_isAntidiagonalTermOrder`.
- Concrete reducedness corollary:
  `Determinantal.normalizedGrPlusOne_isReduced_antiDiagonalLex`.
- Concrete reduced Gröbner-basis wrapper:
  `Determinantal.normalizedGrPlusOne_isReducedGroebnerBasis_antiDiagonalLex`.
- Original Gröbner-basis paper-number alias:
  `Determinantal.theorem1_GrPlusOne_isGroebnerBasis_of_isAntidiagonalTermOrder`.

## Meaning of the audits

`lake build` performs elaboration and kernel checking. `check_sources.sh`
scans tracked Lean sources for `sorry`, `admit`, or `sorryAx`, and for axiom or
unsafe declarations, including declarations marked `private` or `protected`.
`check_axioms.py` compares `#print axioms` reports with an explicit
foundational allowlist. `PaperAudit.lean` checks manuscript-facing names and
types. These checks establish different properties; none replaces the others
or independently validates the manuscript’s prose.

## Metadata generation

On a clean committed revision, run:

```bash
python3 scripts/artifact_metadata.py --require-clean --format markdown
```

## Fixed source links

After committing the frozen revision, run:

```bash
python3 scripts/paper_links.py --format markdown
```

The generator refuses a dirty working tree because its links refer to `HEAD`.

## Trusted base

The trusted base consists of the Lean kernel and the pinned imported
libraries. Mathlib provides the general algebraic infrastructure;
`groebner_proj` provides the Gröbner-basis framework. The axiom audit currently
allows the foundational constants `propext`, `Classical.choice`, and
`Quot.sound`; it rejects `sorryAx` and any unlisted axiom.

## Mutable branch and frozen artifact

The `main` branch is mutable. No immutable manuscript artifact or archival DOI
is claimed here. The final paper must cite an immutable commit/tag and an
archival DOI or equivalent persistent identifier, not changing `main`.

Repository-side checks cannot establish manuscript-only AFM requirements. The
authors must still deposit the manuscript in an accepted open archive, insert
fixed links for the main results, declare funding and conflicts, and complete
the archival steps in the [release checklist](docs/release_checklist.md).
