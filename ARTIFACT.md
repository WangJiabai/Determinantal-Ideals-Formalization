# AFM Artifact Guide

## Artifact scope

This artifact formalizes the Gröbner-basis component of Sturmfels’s theorem
for a generic matrix over an arbitrary field, together with a
coefficient-normalized reduced refinement. The main results apply to every
monomial order satisfying `IsAntidiagonalTermOrder`; `antiDiagonalLex` is a
concrete instance. The original `(r + 1) × (r + 1)` generic minors form a
Gröbner basis and satisfy the interreduction condition. Scaling each minor by
the inverse of its anti-diagonal leading coefficient yields a reduced Gröbner
basis under the Lean library’s monic definition of reducedness. In Lean, `Jr`
is the ideal generated by the original minors. The artifact does not formalize
the rank-variety geometry or the identification of its vanishing ideal with
`Jr`.

## Requirements

- Git;
- elan with Lake and the pinned Lean toolchain;
- network access for cloning the repository and for the initial
  dependency/cache download;
- no external computer-algebra system as a proof oracle.

## Clean-checkout reproduction

Reviewers should reproduce the artifact from the immutable revision cited by
the manuscript. Once that revision has been frozen, begin from a clean
checkout:

```bash
git clone https://github.com/WangJiabai/Determinantal-Ideals-Formalization.git
cd Determinantal-Ideals-Formalization
git checkout ARTIFACT_COMMIT_OR_TAG
```

Replace `ARTIFACT_COMMIT_OR_TAG` with the immutable commit or tag cited by the
manuscript.

Then run:

```bash
lake exe cache get
bash scripts/check_artifact.sh
```

A successful `check_artifact.sh` run ends with
`Artifact checks completed successfully`. The script scans tracked Lean
sources, builds the project, compiles the paper-facing interfaces, enforces the
declared axiom allowlist, verifies the declaration locations used for fixed
links, and reports revision and dependency metadata. The initial cache command
may access the network; the proofs themselves do not call an external
computer-algebra system.

## Paper-facing entry points

- [`PaperAudit.lean`](PaperAudit.lean) compiles manuscript-facing declaration
  references and selected theorem-signature checks.
- [`Audit.lean`](Audit.lean) reports the transitive axiom dependencies of
  selected declarations.
- [`Groebner.lean`](MyProject/Determinantalideals/Groebner.lean) contains the
  Hilbert-function argument and the unnormalized Gröbner-basis theorem.
- [`ReducedGroebner.lean`](MyProject/Determinantalideals/ReducedGroebner.lean)
  contains interreduction, coefficient normalization, and reducedness.
- Main Gröbner-basis theorem:
  `Determinantal.GrPlusOne_isGroebnerBasis_of_isAntidiagonalTermOrder`.
- Interreduction theorem for the original minors:
  `Determinantal.GrPlusOne_isInterreduced_of_isAntidiagonalTermOrder`.
- Divisibility rigidity lemma used by interreduction:
  `Determinantal.antidiagExp_le_permExp_imp_minorIndex_eq`.
- Main reduced-refinement theorem:
  `Determinantal.normalizedGrPlusOne_isReducedGroebnerBasis_of_isAntidiagonalTermOrder`.
- Reduced-refinement paper-number alias:
  `Determinantal.theorem1_normalizedGrPlusOne_isReducedGroebnerBasis_of_isAntidiagonalTermOrder`.
- Concrete reducedness corollary:
  `Determinantal.normalizedGrPlusOne_isReduced_antiDiagonalLex`.
- Concrete reduced Gröbner-basis wrapper:
  `Determinantal.normalizedGrPlusOne_isReducedGroebnerBasis_antiDiagonalLex`.
- Original Gröbner-basis paper-number alias:
  `Determinantal.theorem1_GrPlusOne_isGroebnerBasis_of_isAntidiagonalTermOrder`.

## Meaning of the audits

`lake build` performs elaboration and kernel checking. `check_sources.sh`
performs a source-level scan of tracked Lean files for `sorry`, `admit`, or
`sorryAx`, and for axiom or unsafe declarations, including declarations marked
`private` or `protected`. `Audit.lean` emits the `#print axioms` reports;
`check_axioms.py` runs that audit and compares the reported dependencies with
the explicit foundational allowlist. `PaperAudit.lean` compiles
manuscript-facing declaration references and selected theorem-signature
checks. Each check establishes a distinct property; none replaces the others
or independently validates the manuscript’s prose.

## Metadata generation

On a clean, committed revision, run:

```bash
python3 scripts/artifact_metadata.py --require-clean --format markdown
```

## Fixed source links

After committing the frozen revision, run:

```bash
python3 scripts/paper_links.py --format markdown
```

The generator refuses to run on a dirty working tree because the generated
links refer to `HEAD`.

## Trusted base

The trusted base comprises the Lean kernel and the pinned imported libraries.
Mathlib provides the general algebraic infrastructure, while `groebner_proj`
provides the Gröbner-basis framework. The axiom audit currently permits the
foundational constants `propext`, `Classical.choice`, and `Quot.sound`; it
rejects `sorryAx` and dependencies on any axiom outside the allowlist.

## Mutable branch and frozen artifact

The `main` branch is mutable. No immutable manuscript artifact or archival DOI
is claimed here. The final paper must cite an immutable commit or tag and an
archival DOI or equivalent persistent identifier rather than the mutable
`main` branch.

Repository-side checks cannot verify compliance with manuscript-only AFM
requirements. The author must still deposit the manuscript in an accepted
open archive, insert fixed links for the main results, declare funding and
conflicts, and complete the archival steps in the
[release checklist](docs/release_checklist.md).
